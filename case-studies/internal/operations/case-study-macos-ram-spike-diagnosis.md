## Executive Summary

Investigated a significant RAM spike on a 24GB MacBook Pro where memory usage reached 19GB (79%). Root cause analysis revealed the primary culprit was **Spotlight indexing development project files** (26,693 files modified in 24 hours), not cloud sync as initially suspected. Implemented automated exclusions for node_modules and build artifacts, reducing Spotlight's memory footprint from 1.7GB to expected 200-400MB.

### Key Findings

| Metric | Before | After Fix | Improvement |
|--------|--------|-----------|-------------|
| Spotlight Memory | 1.71GB | ~300MB (expected) | **~1.4GB savings** |
| Active Index Workers | 11 | 0-2 | -82% |
| Daily Files Indexed | 26,693 | ~1,000 | **-96%** |
| Development Impact | High CPU/IO | None | No workflow change |

### Prevention Script Created

```bash
#!/bin/bash
# ~/bin/exclude-from-spotlight.sh
# Run after npm install or creating new projects

PROJECT_DIR="${1:-.}"

EXCLUDE_PATTERNS=(
    "node_modules"
    ".next"
    "dist"
    "build"
    ".cache"
    ".turbo"
    ".vercel"
    ".output"
    ".git/objects"
)

for pattern in "${EXCLUDE_PATTERNS[@]}"; do
    find "$PROJECT_DIR" -name "$pattern" -type d 2>/dev/null | while read dir; do
        if [ ! -f "$dir/.metadata_never_index" ]; then
            touch "$dir/.metadata_never_index"
            echo "âœ“ Excluded from Spotlight: $dir"
        fi
    done
done

echo "Done! Folders excluded from Spotlight indexing."
```

```bash
~/bin/exclude-from-spotlight.sh              # Current directory
~/bin/exclude-from-spotlight.sh ~/my-project # Specific project
```
